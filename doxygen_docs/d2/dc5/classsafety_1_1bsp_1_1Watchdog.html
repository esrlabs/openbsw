◆ executeFastTest()

bool safety::bsp::Watchdog::executeFastTest ( uint32_t  timeout)
static

Execute the WD fast test. On first call this will trigger the test for the low byte, then on second call it will trigger the test for the high byte of the counter. Afterwards it should always return true, indicating that the WD test was successful.

Parameters
timeoutdelay for the watchdog to timeout
Returns
  • false = the check was started, but it was unsuccessful
  • true = the check executed successfully since the last power-on reset, no further check is needed for this cycle
  • does not return = the check was started and watchdog has reset the ECU as expected
