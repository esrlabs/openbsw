Detailed Description

The implementation of ICanTransceiver using Linux SocketCAN stack

The class assumes that all its methods except write(), the constructor, and the destructor are run in the same task context. The deviation from this can result in unobvious UBs. The transceiver state change detection is currently not implemented, the corresponding callback is never called.
